## 👋 About Me

Senior Backend Engineer with **10+ years of experience** building **high-performance, reliable, and compliant backend systems** for data-intensive products. I specialize in **TypeScript-first Node.js architectures**, system scalability, and production readiness.

I enjoy owning services **end-to-end**—from API design and data modeling to observability, security, and operational excellence. My work typically focuses on **order flows, real-time systems, healthcare platforms, and business-critical backends** used by real users at scale.

## 🧠 Recent Projects

### 🚀 QMedica — Nuclear Medicine Reporting Platform  
**Production system used in multiple hospitals**

- **Domain:** Nuclear medicine diagnostic reporting  
- **Users:** Physicians, residents, operators, and patients  
- **Stack:** MERN (TypeScript ≈92%), with supporting C#/.NET, PHP, Lua, CSS  
- **Website:** https://QMedica.ir  

**Technical Highlights**
- TypeScript-first engineering with modular, maintainable architecture  
- Clear separation of concerns and scalable service design  
- Performance-minded async flows and optimized runtime behavior  
- Production-ready error handling, configuration discipline, and extensibility  

> A real-world healthcare system with high user satisfaction and daily operational usage.

---

### 🧩 OpenTask — Full-Stack Task Management Platform  
**Production-grade architecture with reliability focus**

🔗 https://github.com/mohsen-opentask  

**Backend (NestJS / Node.js)**
- Versioned REST API (`/api/v1`) with TypeScript + NestJS  
- PostgreSQL + Prisma (type-safe schema, migrations, optimized queries)  
- JWT authentication with **rotating refresh tokens (DB-backed)**  
- RBAC via guards, rate limiting, strict DTO & env validation  
- Swagger documentation, unit & e2e tests (Jest)

**Frontend (Next.js)**
- TypeScript-first Next.js client with TailwindCSS  
- Protected routing, task CRUD (filtering, sorting, pagination)  
- Shared API request layer and reusable component architecture  

**Reliability Mindset**
- Idempotent task creation using `Idempotency-Key`  
- Offline-first PWA (Service Worker + IndexedDB queue via Web Worker)

### 🎯 Monopoly — Real-Time Multiplayer Platform  
**Low-latency, event-driven system**

- **Backend:** Node.js + NestJS (WebSocket architecture)  
- **Database:** PostgreSQL + Prisma ORM  
- **Caching / PubSub:** Redis  
- **Frontend:** Next.js (TypeScript)

**Highlights**
- Real-time systems design with scalable event handling  
- Session/state coordination via Redis  
- Type-safe backend architecture and database modeling  
- Focus on performance, concurrency, and synchronization
